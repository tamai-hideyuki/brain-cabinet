# 瞬発キャプチャ用テンプレ
```
---
type: scratch
---

【今の一言】
{とにかく一言で}

【気になった理由】
{違和感・学び・迷い}

【使えそう？】
- 使えそう / 微妙 / 今は不明
```


# 学習メモ用リキャップテンプレ

## 目的：
- 学んだ知識を「判断に使える部品」に変換する

## 設計思想

- 要約しない
- 教科書的説明を書かない

### 「いつ使うか / 使わないか」を最優先

```
---
type: learning
title: {未来の自分が検索する言葉}
tags:
  - learning
  - {技術領域}
  - {設計/実装/運用}
---

## テーマ

## 解決する問題

## 使う場面（When）

## 使わない場面（Avoid）

## 判断に使うときの一言


## 関連・派生

```

```
---
type: learning
title: {未来の自分が検索する言葉}
tags:
  - learning
  - {技術領域}
  - {設計/実装/運用}
---

## テーマ
{何についての知識かを一言で}

## 解決する問題
{この知識が「何に効くのか」}

## 使う場面（When）
- {具体的な状況}
- {判断が必要になるタイミング}

## 使わない場面（Avoid）
- {適用すると逆効果なケース}
- {オーバーエンジニアリングになる状況}

## 判断に使うときの一言
> {判断時にそのまま脳内再生される短文}

## 関連・派生
- {関連する設計/技術}

```

### 良い例（超短縮）
```
## テーマ
DTOとEntityの分離

## 解決する問題
ドメインモデルの汚染・責務肥大

## 使う場面
- API境界
- 外部I/Oが入る層

## 使わない場面
- 超小規模CRUD
- 変更頻度が極端に低い場合

## 判断に使うときの一言
> 外と話すならDTO、内はEntityを守る
```

# 判断メモテンプレ

## 目的：
- 経験を「再生可能な判断」に変換する

### これが溜まり始めた瞬間、
- あなたの成長速度は 非線形 になります。

```
---
type: decision
title: {◯◯の場合、どう判断したか}
tags:
  - decision
  - {技術領域}
  - {プロジェクト/文脈}
---

## 状況（Context）
- 何が起きていたか
- 制約（時間・人・既存設計）
- なぜ判断が必要だったか

## 選択肢（Options）
- 案A：
- 案B：
- （あれば）案C：

## 判断（Decision）
{最終的に選んだ判断}

## 理由（Reason）
- なぜこれを選んだか
- 何を捨てたか（トレードオフ）

## 結果（Result）
- どうなったか
- 後から見てどうか（未記入OK）

## 参照した知識（References）
- [[学習メモID or タイトル]]
- [[過去の判断メモ]]

```

### 良い例

```
## 状況
APIレスポンスに直接Entityを返していた

## 選択肢
- Entityをそのまま返す
- DTOを導入する

## 判断
DTOを導入する

## 理由
- 将来の仕様変更耐性
- ドメイン保護を優先

## 参照した知識
- DTOとEntityの分離
```
